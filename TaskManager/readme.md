# TaskManager – Менеджер задач с графическим интерфейсом

## Описание проекта

TaskManager – это десктопное приложение для управления задачами, разработанное на C++ с использованием фреймворка Qt. Программа предоставляет удобный интерфейс для создания, отслеживания и организации задач с установкой сроков выполнения и приоритетов. Приложение демонстрирует практические навыки разработки кроссплатформенного ПО с современными подходами к архитектуре.

## Основные возможности

- Создание, редактирование и удаление задач с названием, описанием и сроком выполнения
- Установка приоритета задач по 5-балльной шкале
- Визуальная индикация статуса задач через цветовое кодирование:
  - Зеленый – выполненные задачи
  - Серый – архивированные задачи
  - Розовый – просроченные задачи
  - Белый – активные задачи
- Автоматическая фоновая проверка приближающихся дедлайнов
- Уведомления о задачах, у которых срок выполнения наступит в течение часа
- Сохранение размера и положения окна между сеансами работы
- Работа с локальной базой данных SQLite для хранения задач

## Технологии

- Язык программирования: C++11/14
- Фреймворк: Qt 5 (Core, Gui, Widgets, Sql)
- База данных: SQLite через QtSql
- Система сборки: CMake 3.5+
- Многопоточность: QThread для фоновых операций
- Архитектура: объектно-ориентированный дизайн

## Что демонстрирует этот проект

### Практические навыки C++/Qt

- Разработка кроссплатформенных GUI-приложений с использованием Qt Widgets
- Работа с базами данных через QtSql (CRUD-операции, подготовленные запросы)
- Реализация многопоточности с помощью QThread для фоновой задачи проверки "Дедлайна задачи"
- Использование сигналов и слотов для взаимодействия между компонентами
- Работа с QTimer для периодических операций
- Сохранение и восстановление состояния приложения через QSettings
- Создание диалоговых окон и пользовательских форм

### Архитектурные подходы

- Разделение ответственности между классами (DatabaseManager, DeadlineChecker, MainWindow)
- Инкапсуляция логики работы с БД в отдельный класс
- Использование паттерна "Наблюдатель" через сигналы/слоты для уведомлений о дедлайнах
- Чистая компиляция с включением текущего каталога

### Работа с инструментами

- Настройка системы сборки CMake с поддержкой автоматических MOC, UIC, RCC
- Организация проекта для кроссплатформенной разработки
- Использование отладочных инструментов Qt (qDebug)
- Поддержка различных конфигураций сборки (Debug/Release)

## Структура проекта

- CMakeLists.txt – конфигурация сборки CMake
- main.cpp – точка входа приложения
- mainwindow.h/cpp – основное окно приложения
- mainwindow.ui – UI-форма главного окна
- DatabaseManager.h/cpp – менеджер работы с базой данных
- TaskDialog.h/cpp – диалог создания/редактирования задачи
- TaskDialog.ui – UI-форма диалога задачи
- DeadlineChecker.h/cpp – фоновый поток проверки дедлайнов

## Требования для сборки

- Компилятор C++ с поддержкой C++11 (GCC, Clang, MSVC)
- Qt 5 (модули: Core, Gui, Widgets, Sql)
- CMake 3.5 или выше
- SQLite3 (обычно поставляется с Qt)

## Сборка и запуск

```
# Клонирование репозитория
git clone <repository-url>
cd TaskManager

# Создание директории для сборки
mkdir build
cd build

# Генерация проектных файлов
cmake ..

# Сборка проекта
cmake --build . --config Release

# Запуск приложения
./TaskManager  # или TaskManager.exe на Windows
```

## Использование

1. Добавление задачи: Нажмите кнопку "Добавить", заполните название, описание, установите дедлайн и приоритет
2. Отметка выполнения: Выберите задачу и нажмите "Выполнить"
3. Архивирование: Выберите задачу и нажмите "В архив"
4. Удаление: Выберите задачу и нажмите "Удалить"
5. Обновление списка: Кнопка "Обновить" перезагружает список задач

Приложение автоматически уведомит вас о задачах, дедлайн которых наступит в течение часа.

## Особенности реализации

- Безопасная работа с потоками: DeadlineChecker использует QThread с корректным завершением
- Валидация данных: Проверка обязательных полей при создании задач
- Устойчивость к ошибкам: Обработка ошибок БД с выводом диагностических сообщений
- Локализация дат: Отображение дат в удобном для пользователя формате
- Визуальная обратная связь: Цветовое выделение строк в зависимости от статуса задачи
